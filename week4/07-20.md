# be-DailyJob
| 오늘은 개발은 하지 않고 회의만 진행함.

## 회의 요약
* **프로젝트 기본 환경 및 협업 세팅**: GitHub Collaborator 설정, Figma 공용 리포지토리 구성, Branch 전략(dev, main 분리) 수립 완료
* **서비스 핵심 정책 결정**: 
  * 선결제 원칙 도입 (결제 테이블이 배송주문에 1:1로 매핑되는 구조)
  * 전자서명 라이브러리(`signature_pad`) 도입 및 타임스탬프·IP 기반의 위변조 방지 처리 방안 채택
  * 취소/패널티 정책(픽업 전/후, 부르미/드리미 구분) 및 예외 처리 흐름(전문 업체 인계, 반송 프로세스) 정의
  * 회원 탈퇴 시 Soft Delete 기반의 탈퇴 대기 상태 적용
* **시스템 설계**: State Machine 1차 정의 및 데이터 모델링(회원, 배송_주문, 결제, 정산, 주소 등 ERD 작성) 완료

## 회고

### Keep (좋았던 점)
* **명확한 기술 근거 수립**: 결제-주문 구조, 전자서명 법적 효력 이슈, Soft Delete 정책 등 발생 가능한 예외 상황에 대해 논의하고 명확한 도메인 규칙을 정함.
* **상태 흐름 시각화**: State Machine 생성을 통해 단순 매칭 성공 케이스뿐만 아니라 분쟁, 사고(DISPUTED), 전문기사 인계(PARTNER_HANDOFF_PENDING) 등 복잡한 예외 상태를 사전에 도출함.

### Problem (아쉬웠던 점 및 해결 방안)
* **개발 스택 선정 시 제약 사항**: 모바일 웹 앱 기반(React Native/Capacitor 등) 구현 시 라이더의 실시간 위치(GPS) 추적에 기술적 한계가 있음을 인지함.
  * **해결 방안**: 현 단계에서는 스펙 범위를 명확히 조정하여 현실적인 MVP 구현에 집중하기로 합의.
* **데이터 접근 방식 검토 필요**: JPA와 JDBC 사용 사이에서 객체 단위 쿼리 vs 단일 속성 수정 간의 효율성 고민이 발생함.
  * **해결 방안**: JPA 구조 및 ORM 학습을 병행하여 내일 기술 스택을 최종 확정하기로 결정.
