### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: 패키지 분할
- 그렇게 판단한 이유: 처음에 주어진 model, db 등의 계층 이름으로 가면 각 계층의 클래스들이 가지는 의미를 직관적으로 이해하기 어려울 것 같아서
- 고려했지만 선택하지 않은 대안: model, view, controller 3개로만 나누지 않고 http 관련으로 묶고 domain 등으로 따로 묶기
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 비지니스 로직에 맞게 클래스들을 모아둘 수 있음

### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상: 문제 상황을 어떤 자바 예외와 매칭 시켜야 하는지
- 처음 추측한 원인: 애매한건 커스텀 예외로 만들기
- 원인을 확인하기 위해 시도한 방법: 문제 상황이 생길때마다 exception 패키지에 커스텀 예외 계속 만듦
- 실제 원인: 관행적으로 매칭해서 쓰는 예외들이 있음
- 해결 방법: 이미 많이 쓰는 매칭은 따라가기
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: NullPointerException이나 IllegalArgumentException 같은거는 자바 예외로 매칭시켜서 catch 하기
