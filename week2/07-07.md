### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: `HttpResponse` 의 `responseBody` 를 단순히 매개변수 받은거를 출력하기만 하는걸로 추상화한 것
- 그렇게 판단한 이유: 상태 코드에 맞게 response 만들때마다 메소드도 같이 만들면 비효율적
- 고려했지만 선택하지 않은 대안: `response200`, `response500` 처럼 메소드 별도로 생성
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: header만 상태코드에 맞게 추가해주고 `HttpResponseMessage` 객체를 넘기면 이 객체 자체를 출력하므로 `HttpResponse` 가 하나의 책임만 갖게 됨

### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상: 로그인 성공 실패 판별 메소드가 작동 안하고 종료돼버림
- 처음 추측한 원인: 조건이 false로 리턴돼서 그런거라 생각
- 원인을 확인하기 위해 시도한 방법: 테스트코드 만들어보면서 케이스 검증
- 실제 원인: `URILinker.getPath`가 action 판별 로직보다 앞에 있었기 때문
- 해결 방법: action인 경우와 그렇지 않은 경우로 if else를 나눔
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 전체 로직 순서 숙지하면서 코딩
